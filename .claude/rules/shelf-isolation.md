# Règle d'Isolation Shelf - NON-NÉGOCIABLE

## ⛔ INTERDICTION ABSOLUE

**L'AI NE DOIT JAMAIS:**

1. ❌ Consulter 3A-Shelf de sa propre initiative
2. ❌ Suggérer d'importer des patterns/code depuis shelf
3. ❌ Mentionner proactivement le contenu du shelf
4. ❌ Comparer VocalIA avec le shelf sans demande explicite
5. ❌ Proposer de synchroniser avec shelf
6. ❌ Utiliser yalc push/update automatiquement

## ✅ AUTORISÉ UNIQUEMENT

**Sur DEMANDE EXPLICITE de l'utilisateur:**

- Consulter un fichier shelf spécifique demandé
- Comparer versions si l'utilisateur le demande
- Documenter un transfert SI l'utilisateur l'ordonne
- Répondre à une question directe sur le shelf

## Exemples

### ❌ INTERDIT (initiative AI)
```
"J'ai remarqué que JO-AAA a un pattern HITL, voulez-vous l'importer?"
"Le shelf contient une version plus récente de X..."
"Je vais consulter le shelf pour voir si..."
```

### ✅ AUTORISÉ (demande utilisateur)
```
Utilisateur: "Consulte le shelf pour le pattern HITL"
AI: [consulte et répond]

Utilisateur: "Compare BillingAgent shelf vs VocalIA"
AI: [compare et répond]
```

## Principe Fondamental

**ISOLATION PAR DÉFAUT**

VocalIA est un projet INDÉPENDANT.
Le shelf est une RESSOURCE PASSIVE consultée UNIQUEMENT sur demande utilisateur.
L'AI n'a AUCUNE autorité pour initier des interactions cross-projet.
