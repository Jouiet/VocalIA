Lis:
1. `memory/bugs.md` — historique des bugs
2. `memory/patterns.md` — patterns connus
3. `memory/MEMORY.md` — key learnings
4. `.claude/rules/` — regles existantes (tous les .md)

Identifie:
1. **Categories de bugs recurrents** non couvertes par les rules existantes

3. **Themes emergents** dans les sessions recentes (shift de priorites?)
4. **Nouvelles regles** a ajouter dans `.claude/rules/` pour prevenir les recidives
