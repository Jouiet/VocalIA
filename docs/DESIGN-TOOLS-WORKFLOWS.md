## 1. Google Stitch (UI Generation)

### What It Is
Google Stitch est un outil AI de Google Labs qui génère des interfaces utilisateur complètes à partir de prompts textuels ou images. Basé sur Gemini 2.5/3.

### Access
- **URL**: https://stitch.withgoogle.com/
- **Local API**: `core/stitch-api.cjs` (MCP wrapper)
- **Auth**: Google Account + gcloud config

### Use Cases VocalIA

| Use Case | Mode | Prompt Example |
|:---------|:-----|:---------------|
| Landing page section | Standard | "SaaS pricing table, 3 tiers, dark mode, teal accent" |
| Dashboard widget | Standard | "Analytics card with line chart, KPIs, glassmorphism" |
| Mobile app screen | Pro Screen | "Voice AI call interface, waveform, recording button" |
| Wireframe → UI | Pro Screen | Upload wireframe + "Modern tech startup style" |

### Workflow: Generate UI Component

```bash
# 1. List existing projects
node core/stitch-api.cjs list

# 2. Create project
node core/stitch-api.cjs create "VocalIA Dashboard"

# 3. Generate screen (30-60 seconds)
node core/stitch-api.cjs generate <projectId> \
  "Modern analytics dashboard card with real-time call metrics, dark mode, teal accent #0d9488" \
  DESKTOP \
  GEMINI_3_PRO

# 4. Download HTML/CSS
node core/stitch-api.cjs download <projectId> <screenId>
```

### Best Practices
1. **Start broad, then refine** - "Dashboard" → "Call metrics card" → "Add comparison arrows"
2. **One change at a time** - Avoid compound edits
3. **Specify colors with hex** - "#0d9488" not "teal"
4. **Use Standard Mode** for speed (350 gen/month), Pro for quality (200 gen/month)

### Limitations
- No direct Tailwind export (HTML/CSS only)
- Best for ideation (first 10%), Figma for polish
- Limited to web UI (no native mobile)

- [Codecademy Tutorial](https://www.codecademy.com/article/google-stitch-tutorial-ai-powered-ui-design-tool)
- [Google Developers Blog](https://developers.googleblog.com/stitch-a-new-way-to-design-uis/)

## 2. Google Whisk (Image Generation)

### What It Is
Whisk est un outil AI de Google Labs qui génère des images à partir d'**images de référence** (pas de texte). Utilise Gemini pour captionner + Imagen 3 pour générer.

### Access
- **URL**: https://labs.google/fx/tools/whisk
- **Availability**: 100+ pays (US first)
- **Pricing**: Free (Google Labs), quotas via Google One AI Premium

### Use Cases VocalIA

| Use Case | Subject | Scene | Style |
|:---------|:--------|:------|:------|
| Hero illustration | VocalIA logo | Office environment | 3D isometric |
| Social media | Product screenshot | Gradient background | Flat design |
| Mascot variations | Character image | Different poses | Consistent style |
| Marketing visuals | Product demo | Tech environment | Photorealistic |

### Workflow: Create Marketing Visual

```
1. Open https://labs.google/fx/tools/whisk

2. Drag images for 3 slots:
   - SUBJECT: Upload VocalIA product screenshot
   - SCENE: Upload modern office/tech environment
   - STYLE: Upload reference (e.g., Stripe marketing visual)

3. Click Generate

4. Refine via chat:
   - "Make it more futuristic"
   - "Add teal glow effect"
   - "Remove text, keep illustration only"

5. Download variations you like
```

### Best Practices
1. **Rapid ideation tool** - Not for pixel-perfect edits
2. **Use for concept exploration** - Dozens of options quickly
3. **Good for**: Digital plushies, enamel pins, stickers, concept art
4. **Final polish**: Export to Photoshop/Figma

### Limitations
- No text control (captions are auto-generated by Gemini)
- Output style varies based on inputs
- Not for precise brand compliance

## 3. Remotion (Programmatic Video)

### What It Is
Remotion permet de créer des vidéos avec React. Chaque frame est un composant React, rendu en MP4/WebM.

### Access
- **Install**: `npm init video`
- **Preview**: `npm run dev` → http://localhost:3000
- **Render**: `npx remotion render <composition> out/video.mp4`

### Use Cases VocalIA

| Use Case | Complexity | Template |
|:---------|:-----------|:---------|
| Product demo | Medium | template-helloworld |
| Feature showcase | Medium | Custom + animations |
| Onboarding video | High | Multiple sequences |
| Social ads | Low | reactvideoeditor/remotion-templates |
| 3D product | High | template-three (React Three Fiber) |

### Workflow: Create Marketing Video with Claude Code

```bash
# 1. Create Remotion project
npx create-video@latest vocalia-demo
cd vocalia-demo

# 2. Install Skills (enables Claude Code integration)
npx skills add remotion-dev/skills

# 3. Install dependencies
npm install

# 4. Start preview server
npm run dev

# 5. In another terminal, start Claude Code
claude

# 6. Prompt Claude:
"Create a 15-second VocalIA product video:
- Title slide: 'VocalIA - Voice AI Platform'
- Feature 1: '5 Languages Support' with flag icons
- Feature 2: '$0.06/min' with animated counter
- CTA: 'Start Free Trial'
- Colors: Deep Teal #0d9488, Dark #0f172a
- Font: Inter
- Style: Modern, glassmorphism, subtle animations"

# 7. Render final video
npx remotion render Main out/vocalia-demo.mp4
```

### Key React Hooks

```javascript
import { useCurrentFrame, useVideoConfig, spring, interpolate } from 'remotion';

// Get current frame number
const frame = useCurrentFrame();

// Get video properties
const { fps, durationInFrames, width, height } = useVideoConfig();

// Natural motion
const scale = spring({ frame, fps, config: { damping: 10 } });

// Value mapping
const opacity = interpolate(frame, [0, 30], [0, 1]);
```

### Best Practices
1. **Use Claude Code + Remotion Skills** for fastest iteration
2. **Preview constantly** - `npm run dev` shows live changes
3. **Sequences for organization** - Break video into logical parts
4. **Use springs for animations** - More natural than linear

### Limitations
- Learning curve for React/TypeScript
- Rendering time increases with complexity
- Commercial license for teams >3 employees

- [Remotion Docs - Claude Code](https://www.remotion.dev/docs/ai/claude-code)
- [GitHub - remotion-dev/remotion](https://github.com/remotion-dev/remotion)
- [Free Templates](https://github.com/reactvideoeditor/remotion-templates)

## 4. Gemini MCP (Design Consultation)

### What It Is
Gemini via MCP permet de consulter l'IA pour des décisions de design, générer des palettes, auditer l'accessibilité.

### Access
- **MCP Tools**: `gemini_chat`, `gemini_deep_research`, `gemini_list_models`
- **Config**: `.mcp.json` → gemini server

### Use Cases VocalIA

| Use Case | Tool | Prompt Pattern |
|:---------|:-----|:---------------|
| Color palette | gemini_chat | "Generate a 10-shade palette for enterprise Voice AI SaaS..." |
| Design review | gemini_chat | "Review this CSS for WCAG 2.1 AA compliance..." |
| Trend research | gemini_deep_research | "Research 2026 SaaS design trends from Stripe, Linear, Notion..." |
| Component specs | gemini_chat | "Design a spec for a glassmorphism card component..." |

### Workflow: Generate Design System

```javascript
// Via MCP tool call
{
  "tool": "gemini_chat",
  "prompt": `Design a complete color system for VocalIA Voice AI Platform.

Requirements:
- Primary: Deep Teal for trust & innovation
- WCAG 2.1 AA compliant on #0f172a background
- 10-shade palette (50-950)
- Include semantic colors (success, warning, error, info)

Output format:
- CSS custom properties
- Hex codes with contrast ratios
- Usage guidelines for each shade`
}
```

### Best Practices
1. **Be specific with hex codes** - "#0d9488" not "teal"
2. **Request contrast ratios** - For accessibility verification
3. **Ask for reasoning** - "Why this color conveys trust?"
4. **Use deep_research for trends** - Multi-step investigation

### Prompt Templates

**Color Palette Generation:**
```
Generate a {n}-shade color palette for {brand type}.
Primary hex: {#hex}
Must work on: {background color}
Contrast requirement: WCAG {level}
Include: {semantic colors list}
Output: CSS custom properties with comments
```

**Design Audit:**
```
Audit this {component type} for:
- Color contrast (WCAG 2.2 AA)
- Keyboard navigation
- Focus states
- ARIA labels
Return: Table with issues, severity, fix suggestions
```

## 5. Playwright MCP (Visual Testing)

### What It Is
Playwright MCP connecte les AI agents au browser pour l'automatisation et le testing visuel. Utilise l'accessibility tree (pas les screenshots).

### Access
- **MCP Tools**: `mcp__playwright__*` (navigate, screenshot, snapshot, click, etc.)
- **Config**: `.mcp.json` → playwright server

### Use Cases VocalIA

| Use Case | Tools | Purpose |
|:---------|:------|:--------|
| Visual regression | screenshot + compare | Before/after design changes |
| Accessibility audit | snapshot | DOM structure, ARIA labels |
| E2E design flow | navigate + click + screenshot | User journey verification |
| Responsive testing | resize + screenshot | Multi-device validation |

### Workflow: Visual Verification

```javascript
// 1. Navigate to page
mcp__playwright__browser_navigate({ url: "http://localhost:8080/" })

// 2. Take full-page screenshot
mcp__playwright__browser_take_screenshot({
  filename: "homepage-teal.png",
  fullPage: true
})

// 3. Get accessibility snapshot
mcp__playwright__browser_snapshot()

// 4. Test responsive
mcp__playwright__browser_resize({ width: 375, height: 812 }) // iPhone X
mcp__playwright__browser_take_screenshot({ filename: "homepage-mobile.png" })
```

### Device Emulation
Playwright supports 143 devices including:
- iPhone 12/13/14/15 Pro Max
- iPad Pro
- Pixel 7
- Galaxy S21
- Desktop browsers (1920x1080, 1440x900, etc.)

### Best Practices
1. **Use snapshot for accessibility** - Structured DOM, not pixels
2. **Test key breakpoints** - 375px, 768px, 1024px, 1440px
3. **Automate regression tests** - Before/after CSS changes
4. **Combine with Chrome DevTools** - For CSS debugging

## 6. Chrome DevTools MCP (CSS Debugging)

### What It Is
Chrome DevTools MCP donne à Claude Code des "yeux" dans le browser - inspection DOM, CSS, console, network en temps réel.

### Access
- **Install**: `claude mcp add chrome-devtools npx chrome-devtools-mcp@latest`
- **Requires**: Node.js 22+, Chrome browser
- **MCP Tools**: `mcp__chrome-devtools__*`

### Use Cases VocalIA

| Use Case | Tools | Purpose |
|:---------|:------|:--------|
| CSS layout debugging | take_snapshot + inspect | Fix overflow, alignment |
| Console error analysis | get_console_message | Debug JS errors |
| Network monitoring | list_network_requests | API call inspection |
| Performance profiling | performance_start_trace | Identify bottlenecks |

### Workflow: Debug CSS Issues

```javascript
// 1. Connect to running Chrome
mcp__chrome-devtools__list_pages()

// 2. Select the page
mcp__chrome-devtools__select_page({ pageId: "..." })

// 3. Take DOM snapshot
mcp__chrome-devtools__take_snapshot()

// 4. Inspect specific element
mcp__chrome-devtools__evaluate_script({
  script: "document.querySelector('.glass-panel').getBoundingClientRect()"
})

// 5. Check computed styles
mcp__chrome-devtools__evaluate_script({
  script: "getComputedStyle(document.querySelector('.btn-primary'))"
})
```

### Best Practices
1. **Reduce debugging time by 40%** - No context switching
2. **Use for live CSS fixes** - See changes immediately
3. **Automate accessibility audits** - ARIA, focus, contrast
4. **Monitor console** - Catch errors before users

## Integrated Workflow: Complete Design Cycle

### Phase 1: Ideation (10%)
```
1. Whisk → Generate concept visuals
2. Gemini → Define color palette & specs
3. Stitch → Generate UI components
```

### Phase 2: Implementation (70%)
```
4. Export Stitch HTML → Convert to Tailwind
5. Edit input.css → Add design tokens
6. npm run build:css → Generate utilities
```

### Phase 3: Verification (15%)
```
7. Playwright → Visual regression tests
8. Chrome DevTools → CSS debugging
9. Gemini → Accessibility audit
```

### Phase 4: Content (5%)
```
10. Remotion → Product demo videos
11. Whisk → Social media visuals
```
